# Disk Usage Report Utility
[![.github/workflows/gradle.yml](https://github.com/Daria-Gavrilovaa/Ipprpo3/actions/workflows/gradle.yml/badge.svg)](https://github.com/Daria-Gavrilovaa/Ipprpo3/actions/workflows/gradle.yml)

**Автор:** Дарья Гаврилова  
**Группа:** ИТИ-31 
**Вариант:** 4

Это простая программа, которая проверяет свободное место на жестких дисках вашего компьютера. Она выдаёт результат в удобном компьютерном формате (JSON).

**Для чего это нужно:**
Программа помогает быстро понять:
1. Какие диски переполнены (заняты больше, чем вы разрешили).
2. Сколько всего памяти есть на компьютере и сколько из неё занято.

**Технологии:**
Проект написан на языке **Java (версия 17)**. Для автоматизации процессов используется инструмент **Gradle**.

---

## Пример работы

Программа работает в текстовом режиме (через консоль).

**Что происходит внутри (Входные данные):**
Программа сама запрашивает у системы информацию.
*   *Windows:* Запускает PowerShell команду для получения списка дисков.
*   *Linux/macOS:* Запускает команду `df -h`.

**Что вы видите на экране (Выходные данные):**
Программа выводит отчет в формате JSON.

Пример отчета, если диск `C:` переполнен (занят на 95%):

```json
{
  "threshold_exceeded": [
    {
      "filesystem": "C:",
      "total": "500G",
      "used_percent": 95
    }
  ],
  "summary": {
    "total_gb": 597,
    "total_used_gb": 520
  }
}
threshold_exceeded — список дисков, где превышен лимит.
summary — общая сводка по всем дискам (Всего ГБ / Занято ГБ).
# Предварительная настройка (Конфигурация)
Перед запуском вы можете настроить параметры работы программы. Настройки находятся в следующем файле:
DiskUsageReport\src\main\resources\app.properties.
Файл можно открыть любым текстовым редактором (Блокнот, Notepad++). Для этого необходимо нажать на файл правой кнопкой мыши и выбрать "открыть с помощью".
Содержимое файла и значение параметров:
# Порог использования в %.
# Если диск заполнен больше, чем на это число, он попадет в отчет.
# Например, 90 означает 90%.
usage.threshold.percent=90

# Единицы измерения для отчета.
# В данной версии программы всегда используется GB, параметр оставлен для будущих обновлений.
output.units=GB
Совет: Если вы хотите протестировать отчет, установите значение usage.threshold.percent=10, чтобы в отчет попали почти все диски.
## Как выполнить сборку проекта
Сборка — это процесс превращения исходного кода в готовую программу.
Требования:
На компьютере должна быть установлена Java JDK 17 или новее.
#Инструкция для Windows:
-Откройте папку с проектом.
-Нажмите правой кнопкой мыши на пустом месте "Открыть в терминале" (или PowerShell).
-Введите команду:
.\gradlew.bat build
-Дождитесь надписи BUILD SUCCESSFUL.
#Инструкция для Linux / macOS:
-Откройте терминал в папке проекта.
-Введите команду:
./gradlew build
-Дождитесь надписи BUILD SUCCESSFUL.
Готовый файл программы (disk-usage-report.jar) появится в папке build/libs/.
## Как запустить тесты
Тесты — это автоматическая проверка того, что программа работает правильно и не содержит грубых ошибок.
Введите в терминале команду
# Windows
.\gradlew.bat test

## Linux / macOS
./gradlew test

Ожидаемый результат:
Если всё хорошо, вы увидите сообщение BUILD SUCCESSFUL.
Если тесты нашли ошибку, появится сообщение BUILD FAILED, и программа укажет, что именно сломалось.
# Как запустить приложение
Введите команду:
# Windows
.\gradlew.bat run

# Linux / macOS
./gradlew run
После выполнения команды вы увидите JSON-отчет в окне терминала.
